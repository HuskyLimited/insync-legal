Privacy Policy

Insync Post · Last updated: April 2026

Insync Post is a single-user desktop application for scheduling video posts to the operator's own TikTok creator account. This policy describes what data the application processes and how.

1. What data the application stores

  • OAuth tokens. When you authenticate your TikTok account, the access token and refresh token returned by TikTok's OAuth endpoint are stored as a JSON file on your local machine (backend/.secrets.json). They are not transmitted anywhere except directly to TikTok's official API endpoints.
  • Video files. Video clips you import are stored in a local folder (backend/storage/clips/) on your machine. They are uploaded directly to TikTok via the Content Posting API at the scheduled time. They are not sent to any other server.
  • Local metadata. Clip metadata (filename, duration, dimensions), schedule entries, and composer drafts are stored in a local SQLite database on your machine.
  • Engagement data. If you authorize Display API access, the application reads your own past post statistics (likes, views, post times) directly from TikTok and stores aggregated time-of-day analytics locally.

2. What data the application does not collect

  • The application has no servers, no analytics, no telemetry, and no third-party integrations besides TikTok itself.
  • No personal information beyond your TikTok username (visible in the app UI) is processed.
  • No information about other TikTok users is read or stored.

3. Sharing

The application does not share any data with any third party. Communication is exclusively between your machine and TikTok's official API endpoints.

4. Retention

All stored data lives on your local machine and remains until you delete it. There is no remote retention.

5. Disconnecting

You can disconnect your TikTok account at any time from within the application; this deletes the local OAuth tokens file. You can also revoke the application's access from your TikTok account settings.
